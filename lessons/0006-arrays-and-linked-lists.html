Arrays & Array vs Linked List

16th-May: "Focus on the data structure part, the types… specifically the array, because the array is the basic one." The array-vs-linked-list comparison is a classic short-answer. This is the foundation everything else sits on.

1. What an array is

  • A collection of homogeneous elements (all same type) in contiguous memory.
  • Index starts at 0. For size n, the last valid index is n − 1.
  • Static: size is fixed at declaration. This is the array's main limitation.
int a[5];          // indices 0,1,2,3,4 — five slots, fixed
That "n − 1" fact is everywhere: stack-full is top == MAX-1, binary search uses U = n-1, loops run i < n. Know it cold.

2. Array operations

OperationWhat happens
Access a[i]Direct — O(1). No looping.
TraverseVisit each element once with a for loop — O(n).
Insert at positionShift later elements right to make space — O(n).
Delete at positionShift later elements left to fill the gap — O(n).
// insert value v at index p in array of length n
for(i = n; i > p; i--) a[i] = a[i-1];   // shift right
a[p] = v;
n++;

3. Linked list — the contrast

A linked list stores data in nodes scattered in memory; each node holds data + a pointer to the next node. First node = start, last node's pointer = NULL.

struct Node {
    int data;
    struct Node *next;   // address of the next node
};
  • Why struct? It bundles heterogeneous data — the value AND the pointer.
  • Why a pointer? A pointer is the only variable that holds a memory address (the "link").
Lecturer's analogy: a train. Coach positions (A1, A2…) look adjacent, but coach numbers (the real memory addresses) are scattered — couplings (pointers) link them in order.

4. Array vs Linked List (memorise this table)

ArrayLinked List
MemoryContiguous (adjacent)Scattered, joined by pointers
SizeFixed (static)Grows/shrinks (dynamic)
Access by indexDirect, O(1)Must walk from start, O(n)
Insert / deleteShift elements, O(n)Re-point a link, O(1) at a known node
Extra memoryNone (data only)One pointer per node

Linked-list types (SLM Module 3): singly, doubly, circular, circular-doubly. Operations: create, traverse, search, insert, delete, sort.

5. Scoring in the exam

"Differentiate array and linked list" → give the table above (memory, size, access, insert/delete). Four crisp rows beat a vague paragraph. For a definition, lead with "contiguous + homogeneous + static" for arrays.
