Linear & Binary Search + Complexity

From the 16th-May revision class: "In your exam you can only expect the linear search and the binary search." Plus the complexity values feed Section-A MCQs — and almost nobody in your class had read that chapter. This lesson nails both.

1. Linear (Sequential) Search

Check each element one by one from index 0 until you find the key (or run out). Works on any array — sorted or not.

int linearSearch(int a[], int n, int key){
    int i, flag = 0;                  // flag optional
    for(i = 0; i < n; i++){
        if(a[i] == key){ flag = 1; break; }  // found → stop
    }
    if(flag == 0) printf("Not found");
    else          printf("Found at index %d", i);
}

Complexity: best O(1) (key is first), worst & average O(n) (key last or absent).

2. Binary Search

Pre-condition: the array MUST be sorted. If they hand you an unsorted array, binary search is invalid — say so.

Repeatedly halve the range. L = first index (0), U = last index (n−1), mid = (L + U) / 2.

Compare key with a[mid]Action
key == a[mid]Found — stop.
key < a[mid]Search left half → U = mid − 1
key > a[mid]Search right half → L = mid + 1
L > URange empty → not found.
int binarySearch(int a[], int n, int key){
    int L = 0, U = n - 1, mid;
    while(L <= U){
        mid = (L + U) / 2;
        if(a[mid] == key){ printf("Found at %d", mid + 1); return mid; }
        else if(a[mid] < key) L = mid + 1;   // go right
        else                  U = mid - 1;   // go left
    }
    printf("Not found");                      // L > U
    return -1;
}
Write the iterative version above. The teacher said binary search with recursion will NOT be asked (that's the MCA syllabus).

3. Complexity values (memorise for MCQs)

AlgorithmBestWorst / Average
Linear searchO(1)O(n)
Binary searchO(1)O(log n + 1)
Array access by indexO(1)O(1)

Why binary beats linear: halving the range each step gives a logarithmic count of comparisons instead of linear. That's the whole point of sorting first.

Typical MCQ: "Worst-case complexity of binary search?" → O(log n + 1). "Of linear search?" → O(n).
