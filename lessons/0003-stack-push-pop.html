Stack push & pop in C

From the 12th-April class: "They may ask you to write a code where you are inserting the data into the stack → write the push code. Delete an element → write the pop code." He also promised: "the question will be pretty easy." This is TIER 1 — learn both functions cold.

1. The mental model (LIFO)

A stack is a pile of plates. You add and remove only from the top. Last In, First Out. Only two operations matter:

  • push = insert an item on top
  • pop = remove the top item

An array-based stack needs just two things: the array, and an integer top that holds the index of the current top element.

#define MAX 5      // capacity (array is static — size fixed)
int stack[MAX];
int top = -1;      // -1 means EMPTY (no valid index yet)
Why top = -1 for empty? Array indices start at 0. If the stack held one item it would sit at index 0. So "nothing yet" must be one below zero → −1. This single fact powers both checks below.

2. The two guard conditions

ConditionMeaningTest
Overflowpush onto a FULL stacktop == MAX - 1
Underflowpop from an EMPTY stacktop == -1

For MAX = 5, valid indices are 0,1,2,3,4. So full = top at 5 − 1 = 4. Always check the guard first, before touching the array.

3. push() — insert

void push(int item){
    if(top == MAX - 1){            // 1. overflow guard
        printf("Stack Overflow");
        return;
    }
    top = top + 1;                 // 2. move top up
    stack[top] = item;             // 3. place the item
}

Order matters: check → increment → assign. Increment before assigning, so top points at the new slot.

4. pop() — delete

int pop(){
    if(top == -1){                 // 1. underflow guard
        printf("Stack Underflow");
        return 0;
    }
    int item = stack[top];         // 2. read the top item
    top = top - 1;                 // 3. move top down
    return item;                   // 4. hand it back
}

Mirror of push: check → read → decrement. We don't erase the old slot — moving top down makes it unreachable, which is enough.

6. Linked-list version (1 line to remember)

If they ask for the linked-list stack, the empty test changes from top == -1 to top == NULL. Push = new node points to old top, top = new node. No fixed size (dynamic). The teacher called the linked-list stack "comparatively easier."

7. Scoring this in the exam

Write both functions, keep the overflow/underflow guards (examiners look for them), and add one comment per line saying what it does. That comment line = the "explanation" that lifts you from ~5 to ~7 marks. Watch your semicolons — a missing one loses marks on a code answer.
