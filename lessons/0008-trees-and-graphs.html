Trees, BST & Spanning Trees

Case studies come from "tree traversal" and "shortest-distance algorithms — Prim's, Kruskal's" (12th & 16th). The revision class ran out of time on trees, so this leans on the SLM. Traversals are the highest-value exam skill here.

1. Terminology

  • Root — the top node (no parent). Leaf — a node with no children.
  • Parent / Child — a node directly above / below another.
  • Non-linear, hierarchical data structure (vs the linear array/stack/queue).
  • Binary tree — each node has at most two children (left, right).

2. Binary Search Tree (BST)

BST property: for every node, all values in its left subtree are smaller, all values in its right subtree are larger. This makes search O(log n) on a balanced tree.

We'll use this BST throughout:

50 / \ 30 70 / \ / \ 20 40 60 80
  • Search: start at root; go left if key < node, right if key > node. Same idea as binary search.
  • Insert: search for the key; place the new node where the search falls off.
  • Delete: three cases — leaf (just remove), one child (link child up), two children (replace with inorder successor).

3. Tree Traversals (learn all three)

TraversalOrderMnemonic
InorderLeft, Node, RightL N R
PreorderNode, Left, RightN L R
PostorderLeft, Right, NodeL R N

For the BST above:

  • Inorder: 20 30 40 50 60 70 80  — note: a BST's inorder is always sorted.
  • Preorder: 50 30 20 40 70 60 80
  • Postorder: 20 40 30 60 80 70 50
void inorder(struct Node *t){
    if(t != NULL){
        inorder(t->left);        // L
        printf("%d ", t->data);  // N
        inorder(t->right);       // R
    }
}
Swap the three lines' order to get preorder (print first) or postorder (print last). The recursion shape is identical.

4. Graphs & Spanning Trees (case-study level)

  • Graph — vertices joined by edges. Can be represented by an adjacency matrix or adjacency list.
  • Spanning tree — a tree that connects all vertices of a graph with no cycles.
  • Minimum Spanning Tree (MST) — the spanning tree with the smallest total edge weight. Two algorithms build it:
Prim'sKruskal's
Grow from one vertex; repeatedly add the cheapest edge that reaches a new vertex.Sort all edges; add the cheapest edge that doesn't form a cycle, until all vertices connected.

5. Scoring in the exam

For a traversal question: draw the tree, then write the sequence and name the rule (L-N-R etc.). For a case study, state which algorithm (Prim/Kruskal), then walk the edge selections. Inorder-of-BST-is-sorted is a favourite MCQ.
