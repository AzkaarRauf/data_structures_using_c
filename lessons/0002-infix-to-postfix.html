Infix → Postfix: the one question you will get

Your lecturer said it directly (12th April class): "You will get at least one question from this in your exam." And: "Don't use a shortcut technique — always apply the table." This lesson drills exactly that table method.

1. The three notations

Same expression, three ways to place the operator relative to its two operands:

NotationOperator goes…Example of a + b
Infixbetween operandsa + b
Prefix (Polish)before operands+ a b
Postfix (Reverse Polish)after operandsa b +

Why bother? Computers evaluate postfix easily with a stack — no brackets, no precedence rules needed at run time. Converting infix→postfix is where the stack earns its keep.

2. The precedence table (memorise)

PriorityOperatorsAssociativity
Highest( ) parenthesesLeft → Right
↑^ powerRight → Left
↑* / %Left → Right
↓+ −Left → Right
Lowest= assignmentRight → Left
Key consequence: when * and / meet, the one already in the stack leaves first (left-to-right). That's the move students miss.

3. The algorithm (three columns)

Make a table with columns Scan · Stack · Output. Walk the infix expression left to right:

  1. Operand (A, B, 5…) → write straight to Output.
  2. Operator → before pushing, pop from the stack every operator whose precedence is higher than or equal to the current one (send them to Output). Then push the current operator.
  3. Opening ( → push it. Closing ) → pop to Output until you remove the matching (.
  4. End of input → pop everything left in the stack to Output.

4. Worked example (from your class)

Convert A + B * D / E to postfix.

ScanStackOutputWhy
A(empty)Aoperand → output
++Aoperator → push
B+A Boperand → output
*+ *A B* higher than + → push on top
D+ *A B Doperand → output
/+ /A B D */ equal to * → pop * out, push /
E+ /A B D * Eoperand → output

Answer: A B D * E / +

5. How to score the full 7.5 in the exam

Show the complete table (every scan row), state the precedence rule you used when an operator pops, then box the final postfix string. The table is your explanation — that's what earns marks over a bare answer.

6. Bonus: Infix → Prefix

Same engine, wrapped: (1) reverse the infix, (2) run the postfix table on the reversed string, (3) reverse that result. Done. (Covered in detail in a later lesson.)
