Infix→Prefix, Postfix Evaluation & Recursion

Three TIER-2 items that round out the stack + recursion questions. You already know the infix→postfix table (Lesson 2) — these reuse it.

1. Infix → Prefix (wrap the postfix engine)

SLM method — three steps: (1) Reverse the infix, (2) convert the reversed string to postfix (same table method), (3) Reverse that result → prefix.

One catch when reversing: swap each ( with ) and vice-versa.

Example: A + B * C → prefix

  1. Reverse: C * B + A
  2. Postfix of that: C B * A +
  3. Reverse the result: + A * B C

Answer: + A B C? No — read carefully: reversing C B * A + gives + A * B C. That's the prefix. (Operator before its operands: + joins A and (* B C).)

2. Postfix Evaluation (with a stack)

Scan left→right. Operand → push. Operator → pop the top two, apply (second-popped op first-popped), push the result. At the end, the stack holds the answer.

Evaluate 5 3 + 2 *

ScanStack (bottom→top)Action
55push
35 3push
+8pop 3,5 → 5+3=8 → push
28 2push
*16pop 2,8 → 8*2=16 → push

Result: 16.

3. Recursion

16th-May: "There will be a question on the recursion side… it may be optional." The examiner asks you to write the recursive function only — NOT a whole program. Don't waste time on input/output scaffolding.

A recursive function calls itself, moving toward a base case that stops the recursion.

int fact(int n){
    if(n == 0 || n == 1)   // base case — stops recursion
        return 1;
    return n * fact(n - 1); // recursive call
}

Every recursive function needs two parts: a base case (when to stop) and a recursive call (a smaller sub-problem). Miss the base case → infinite recursion.

int fib(int n){                 // Fibonacci
    if(n <= 1) return n;         // base case
    return fib(n-1) + fib(n-2);  // two recursive calls
}
In your answer, write the 3–4 line function, then one line each: "base case = …", "recursive call = …". That explanation is what earns the marks.
