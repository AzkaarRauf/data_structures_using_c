Introduction to Data Structures

Before writing a single line of C code for a data structure, you need to understand what a data structure is and why it exists. Every exam question on DS ultimately tests this foundation.

1. What is a Data Structure?

A data structure is a way of organising and storing data in memory so it can be accessed and modified efficiently.

Think of it like storage at home. You can throw everything into one big box (an array), hang items on hooks in a line (a linked list), or stack plates (a stack). The structure you choose determines how fast you can find and use things.

Formally, a data structure defines:

  • How data is stored in memory
  • What operations can be performed on it (insert, delete, search, traverse)
  • The relationship between data elements
Exam definition: "A data structure is a particular way of organizing data in a computer so that it can be used efficiently. It defines the relationship between the data and the operations that can be performed on the data."

2. Types of Data Structures

CategoryTypeExamples
PrimitiveBasic built-in typesint, float, char, double
Non-PrimitiveLinearArray, Linked List, Stack, Queue
Non-LinearTree, Graph

Linear vs Non-Linear

  • Linear: Elements arranged in a sequence. Each element has one predecessor and one successor (except first and last). Example: Array, Stack.
  • Non-Linear: Elements arranged in a hierarchy or network. One element can connect to many. Example: Tree, Graph.

Static vs Dynamic

  • Static: Size fixed at compile time. Memory allocated before program runs. Example: Array.
  • Dynamic: Size can grow/shrink at runtime. Memory allocated using malloc()/free(). Example: Linked List.
Common exam question: "Differentiate between linear and non-linear data structures." Give the definition, one example each, and one key difference (sequence vs hierarchy).

3. Abstract Data Type (ADT)

An Abstract Data Type (ADT) describes:

  1. The data stored
  2. The operations on that data
  3. Without specifying how it is implemented

ADT is the what, not the how. A Stack ADT says "push and pop from the top" — it doesn't say whether you use an array or a linked list to implement it.

ADT is like a TV remote's interface: you know pressing "Volume+" increases volume. You don't care if it sends infrared or Bluetooth. The implementation is hidden.
"What is an ADT?" — Answer: It is a logical description of how data is viewed and what operations are allowed, independent of implementation. It provides data abstraction and encapsulation.

4. Why Data Structures Matter — Time Complexity

Different data structures perform operations at different speeds. We measure this using Big O notation — a way to express how performance scales with input size n.

Big ONameExample
O(1)ConstantAccess array[5] directly
O(log n)LogarithmicBinary search
O(n)LinearSearch unsorted array
O(n²)QuadraticBubble sort

Choosing the right data structure is about choosing the right Big O for the operations you need most.

5. Operations on Data Structures

Every data structure supports some set of these operations:

  • Traversal — Visit each element once
  • Insertion — Add a new element
  • Deletion — Remove an element
  • Searching — Find a specific element
  • Sorting — Arrange elements in order
  • Merging — Combine two structures

6. A First Look at C Structures

In C, we use struct to define the building blocks of most data structures:

/* A node — the fundamental unit of linked lists, trees, etc. */
struct Node {
    int data;           /* the value stored */
    struct Node *next;  /* pointer to the next node */
};

You already know struct and pointers — this is all it takes to start building any dynamic data structure in C.
