0. Paper Pattern & How to Answer

Source: 16th May (revision/LMR class) + 12th April

  • Section A — MCQs. Test basics: definitions, complexity values, LIFO/FIFO. E.g. "worst-case complexity of binary search?" → answer O(log n + 1).
  • Section B — Case study + write C code. Each descriptive/code question is 7.5 marks.
  • Questions test fundamental understanding, not deep theory. But framed slightly tricky on purpose ("to confuse you").
Scoring rules the lecturer stated outright:
  • Code + short line-by-line explanation → best score (max ~7, he "never gives 7.5").
  • Code only, no explanation → ~4.5–5 (two marks deducted).
  • Pseudocode WITH explanation → partial marks (won't get zero). Pseudocode without explanation = "nothing."
  • Never leave a blank. Even just header files + basic syntax earns something.
  • A missing semicolon loses marks when a real code answer is expected. Be careful with syntax.
  • Don't overwrite. Align the answer to exactly what's asked.
"Write the code and then write a slight explanation… coding along with a small explanation will be a better approach for tackling the 7.5 marks." — 16th May

Method the teacher taught for any code question: draw the flowchart first, then translate it line-by-line into C. "If my flowchart is clear, I can easily write the code."

Module 1 — Introduction, Algorithms & Arrays

Source: 11th April + 16th May

Data vs Information TIER 2

  • Data = raw facts, figures, values / set of values. Means nothing on its own.
  • Information = data after processing.
Example the teacher uses: "80 marks" alone is data — you can't say pass/fail without the denominator. Once processed → information.

Data Structure — definition & types TIER 1

"A data structure is a way to store and organise data to facilitate the access and modifications." (SLM, Module 1)
  • Primitive data types (NOT data structures): int, float, char.
  • Non-primitive data structures:
    • Linear (data in a sequence): Array, Stack, Queue, Linked List.
    • Non-linear (hierarchical/graph): Tree, Graph, Tables, Sets.
Common student error the teacher called out: answering "int, char, float" when asked "what is a data structure?" — those are data types, not data structures.

Algorithm — definition & characteristics TIER 2

Definition: a step-by-step procedure to solve a given problem (sequence matters — the "making tea" example).

Characteristics per the SLM: Input — zero or more well-defined inputs · Output — one or more well-defined outputs · Finiteness — must terminate · Effectiveness — steps must be basic/doable · Definiteness / Unambiguous — each step clear.
In the 11th-April class the teacher said "an algorithm should have at least one input" and called the slide wrong. The SLM (truth) says "zero or more inputs." In the exam, go with the SLM: zero or more inputs, one or more outputs.

Complexity & Asymptotic Notation TIER 2

Time complexity = how runtime grows with input size n; space complexity = memory used. Expressed with asymptotic notation: Big-O, little-o, Big-Ω (omega), Θ (theta).

In the revision class almost no student had read this chapter — yet the teacher confirmed it IS in the syllabus and feeds MCQs. This is a high-value gap: learn the common complexity values (table below).
AlgorithmBestWorst / Average
Linear searchO(1)O(n)
Binary searchO(1)O(log n + 1)
Array access by indexO(1)O(1)

Arrays TIER 1

"Focus on the data structure part, the types… specifically the array, because the array is the basic one." — 16th May
  • Array = collection of homogeneous elements in contiguous memory.
  • Index starts at 0; for size n, last index = n − 1. (This single fact drives stack-full, binary-search bounds, etc. — know it cold.)
  • Static: size fixed at compile time. This is the array's main limitation vs linked list.
  • Operations: traversal, insertion, deletion, searching. SLM topics: 1-D, 2-D, multi-dimensional arrays, sparse matrix, strings.

Module 2 — Stacks & Queues

Source: 12th April (full class on this) + 16th May

"There will be a lot of questions from the stack in your exam." — 12th April

Stack basics TIER 1

  • LIFO — Last In, First Out (plate-stack analogy).
  • Insertion & deletion happen at the same end = the top.
  • push = insert, pop = delete.
  • Array empty test: top == -1. Linked-list empty test: top == NULL.
  • Overflow = push onto a full stack. Underflow = pop from an empty stack.
  • Stack-full test (array, size MAX): top == MAX - 1.

Push & Pop code TIER 1

"They may ask you to write a code where you are inserting the data into the stack → write the push code. Delete an element → write the pop code." — 12th April. ("Pretty easy" questions.)
#define MAX 5
int stack[MAX];
int top = -1;          // empty

void push(int item){
    if(top == MAX - 1){            // overflow check
        printf("Stack Overflow");
        return;
    }
    top = top + 1;                 // move top up
    stack[top] = item;             // place item
}

int pop(){
    if(top == -1){                 // underflow check
        printf("Stack Underflow");
        return 0;
    }
    int item = stack[top];         // read top
    top = top - 1;                 // move top down
    return item;
}

Infix → Postfix conversion TIER 1

"You will get at least one question from this in your exam." — 12th April
The teacher banned the "shortcut technique" — last semester students lost marks for it. Always use the TABLE method (scan / stack / output columns).

Rules: scan left→right. Operand → send to output. Operator → push to stack, but first pop any stack operator of higher-or-equal precedence to output. At end, pop the rest of the stack to output.

Precedence high→low: ( ) > ^ (power) > * / % > + − > = (assignment). Same-level operators: left-to-right associativity (so * before / when adjacent).

Worked example from class — A + B * D / E → A B D * E / +.

Infix → Prefix conversion TIER 2

SLM steps: (1) Reverse the infix expression, (2) convert the reversed expression to postfix (table method), (3) reverse the result → prefix.

Postfix evaluation TIER 2

Scan left→right; push operands; on an operator, pop the last two operands, apply, push the result. Example: 1 2 3 + * → (2+3)=5 → (1*5)=5.

Queue TIER 3 (concept only — no direct code question)

"There will be no direct question on the Q in your exam… you can't expect any direct question on the programming." — 12th April
  • FIFO — First In, First Out (people in a line).
  • Two ends: rear = insertion (enqueue), front = deletion (dequeue).
  • Both front and rear values increment on their operation.
  • SLM extras: priority queue, circular queue, deque.
Likely Section-A style trap: they give a random sequence of enqueue/dequeue and ask for the output. Practice tracing FIFO by hand. (16th May)

Module 3 — Linked Lists

Source: 19th April + 16th May

Concept TIER 1 (flagged "focus on … the linked list")

  • Linear collection of nodes; each node = data + pointer to the next node.
  • Stored in non-contiguous memory (vs array's contiguous) — the railway-coach analogy: coach position is adjacent, coach number (the memory address) is scattered, a pointer links them.
  • First node = start; last node's pointer = NULL (no next node).
  • Why struct? It stores heterogeneous data — the value AND the pointer. Why pointer? It holds the address of the next node.
ArrayLinked List
MemoryContiguousScattered (linked by pointers)
SizeFixed (static)Dynamic
Access speedFaster (data only)Slower (extra pointer)

Node definition & operations TIER 2

struct Node {
    int data;
    struct Node *next;   // pointer to next node
};

Six operations: create, traverse, search, insert, delete, sort. Insertion can be at beginning, end, or any position.

SLM types: singly, doubly, circular, circular-doubly. Applications: build stacks/queues, polynomial representation.

Module 5 — Searching & Sorting

Source: 16th May (covered in detail) — biggest case-study source

"You can expect [case study] mostly from the searching and sorting part." — 16th May

Linear (Sequential) Search TIER 1

Check each element one by one from index 0 until the key is found. Complexity: best O(1), worst/avg O(n).

for(i = 0; i < n; i++){
    if(a[i] == key){ flag = 1; break; }   // flag is optional
}
if(flag == 0) printf("Not found");
else printf("Found at index %d", i);

Binary Search TIER 1

  • Requires a sorted array. Repeatedly halve the search range.
  • L = 0, U = n − 1, mid = (L + U) / 2.
  • key == a[mid] → found. key < a[mid] → search left (U = mid − 1). key > a[mid] → search right (L = mid + 1). If L > U → not found.
  • Complexity: best O(1), worst O(log n + 1) (an MCQ answer).
L = 0; U = n - 1;
while(L <= U){
    mid = (L + U) / 2;
    if(a[mid] == key){ printf("Found at %d", mid + 1); break; }
    else if(a[mid] < key) L = mid + 1;   // go right
    else                  U = mid - 1;   // go left
}
if(L > U) printf("Not found");
Binary search with recursion will NOT be asked (that's MCA). Write the iterative version above.

Sorting TIER 1

"Go with the quick, merge, or insertion sort only — these are the important ones." — 16th May
  • Insertion sort — build a sorted block, insert each next element into its place (playing-cards analogy). Recommended.
  • Quick sort — partition-exchange. Recommended.
  • Merge sort — divide & merge. Recommended.
  • Selection sort — pick smallest, swap to front (easy "example" sort).

Recursion TIER 2

"There will be a question on the recursion side… it may be optional." — 16th May
  • A function that calls itself (factorial, Fibonacci).
  • The examiner asks you to write the recursive function only — not a full program. Don't waste time writing input/output scaffolding.
  • e.g. factorial: fact(n) = n * fact(n-1) with base case.
